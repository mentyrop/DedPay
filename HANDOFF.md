# DedPay — магазин услуг · хэндофф для Claude Code

Расширение лендинга DedPay до интернет-магазина услуг цифрового содействия.
Дизайн готов к подключению онлайн-кассы: весь путь до оплаты спроектирован,
финальный шаг — в состоянии «касса подключается».

> Позиционирование: DedPay оказывает **услуги цифрового содействия** (оформление
> доступа, продление подписок, пополнение баланса). DedPay **не** продаёт цифровые
> товары и **не** является официальным представителем правообладателей.

## 2. Архитектура файлов

**Стили (CSS-переменные, единая токен-система):**
- `styles.css` — токены, лендинг, общий хедер/футер, кнопки, карта корзины (badge)
- `store.css` — страницы магазина (каталог, карточка, услуга, корзина, checkout, success)
- `legal.css` — юридические страницы

**Данные:**
- `data.js` → `window.DEDPAY` — контент лендинга, навигация, футер, реквизиты
- `store-data.js` → `window.DEDPAY_STORE` — каталог услуг (28 позиций), категории,
  шаблоны текстов (что входит / что нужно / условия / дисклеймер)

**Логика корзины (vanilla, переиспользуема на всех страницах):**
- `cart.js` → `window.DedCart` + `window.fmtPrice`
  - `add(item, qty)`, `setQty(key, n)`, `remove(key)`, `clear()`, `count()`, `total()`, `subscribe(fn)`
  - хранение: `localStorage["dedpay_cart_v1"]`; событие `dedcart:change` + cross-tab `storage`
  - ключ позиции: `id::tier` (одна услуга с разными тарифами = разные позиции)

**React-компоненты (Babel-инлайн, общий global-scope — имена не должны конфликтовать):**
- `components.jsx` → `Icon, Button, Logo, TgGlyph, Eyebrow, SectionHead, Reveal`
- `store-components.jsx` → `useCart, Badge, CartButton, StoreHeader, StoreDrawer, StoreFooter, ServiceCard`
- по странице: `catalog.jsx`, `service.jsx`, `cart.jsx`, `checkout.jsx`, `order-success.jsx`

> ⚠️ Babel-скрипты делят общий scope. Не объявлять повторно `useState/useEffect/useRef` —
> в каждом файле используются уникальные алиасы (`useStateCat`, `useEffectSvc` и т.д.).
> Глобальные объекты стилей называть уникально (никогда `const styles = …`).

## 3. Модель данных услуги (`store-data.js`)

```js
{
  id, slug, category, categoryLabel,
  name, icon,                 // lucide icon name
  type,                       // "Услуга оформления" | "Пополнение"  → бейдж
  tierLabel,                  // "Срок подписки" | "Номинал" | "Объём" | "Формат"
  tagline, desc, term,        // срок выполнения
  tiers: [{ label, price }],  // price = null → "Цена по запросу"
  fromPrice,                  // min(tiers.price) — для карточки
  includes[], needs[], conditions[], disclaimer
}
```

Категории: `music, games, apple, video, cards, other`.
Ссылка из лендинга в каталог с фильтром: `catalog.html?cat=<category>`.

## 4. Ключевые состояния и тексты

- **Карточка услуги** — 2 варианта (`ServiceCard variant="a"|"b"`), переключаются в Tweaks.
  Бейдж «Услуга оформления» / «Пополнение», категория, срок, цена «от …», «Подробнее» + «В корзину».
- **Страница услуги** — выбор тарифа/номинала с пересчётом цены, количество, sticky-панель покупки,
  «Добавить в корзину» + «Задать вопрос в поддержку».
- **Корзина** — позиции с qty-степпером и удалением, итог, заметка про кассу;
  пустое состояние с маскотом и кнопкой «Перейти в каталог».
- **Checkout** — поля: имя/псевдоним*, email*, телефон, комментарий;
  2 чекбокса согласий* (оферта, политика). Клиентская валидация. Кнопка «Оформить заявку».
- **Order-success** — «Заявка создана», номер заявки `DP-XXXXX`, состав заявки,
  3 шага (1 выполнен), CTA «Вернуться в каталог» / «Задать вопрос в поддержку».

Предупреждение про оплату (в корзине, checkout, success):
> «Оплата подключается через онлайн-кассу — оплатить заказ можно будет прямо на сайте.»

## 5. Куда встроить онлайн-кассу

Платёжный шаг намеренно зарезервирован, но не интегрирован. Точка подключения:

`checkout.jsx → submit()`:
1. сейчас: валидирует → пишет заявку в `localStorage["dedpay_last_order"]` → `DedCart.clear()`
   → редирект на `order-success.html?ref=<ref>`.
2. при подключении кассы: после создания заявки на бэкенде вместо редиректа на success —
   редирект на платёжную сессию кассы (ЮKassa/Робокасса/CloudPayments и т.п.);
   success-страница становится callback после успешной оплаты.

Заявка (payload для бэкенда) уже формируется в `submit()`: `{ ref, items, total, count, contact }`.
